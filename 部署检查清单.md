## 🚀 部署步骤

### 1. 创建 GitHub 仓库
```
仓库名：finance-dashboard
可见性：Public
初始化：可以添加 README（可选）
```

### 2. 上传文件到 GitHub

**方式 1：网页上传（推荐）**
1. 打开 GitHub 仓库页面
2. 点击 "Add file" → "Upload files"
3. 拖拽以下文件：
   - app_deploy.py
   - requirements.txt
   - README_GitHub.md（重命名为 README.md）
   - .streamlit/config.toml
4. 点击 "Commit changes"

**方式 2：Git 命令行**
```bash
# 克隆仓库
git clone https://github.com/你的用户名/finance-dashboard.git
cd finance-dashboard

# 复制文件
# 把部署文件复制到该目录

# 提交
git add .
git commit -m "Initial commit: Finance Dashboard v4.0"
git push origin main
```

### 3. 部署到 Streamlit Cloud

1. 访问 https://share.streamlit.io
2. 点击 **"Deploy an app"**（蓝色大按钮）
3. 从 GitHub 选择你的仓库 `finance-dashboard`
4. 配置：
   - **Main file path**: `app_deploy.py`
   - **Branch**: `main`
   - **Python version**: `3.11`（或最新）
5. 点击 **"Deploy!"**

### 4. 等待部署

- 第一次部署需要 **3-5 分钟**
- 可以看到部署日志
- 成功后会显示 "Your app is running!"

### 5. 获取分享链接

部署成功后，链接格式：
```
https://your-username-finance-dashboard-app-deploy-xxxxxx.streamlit.app/
```

**示例**：
```
https://paibwhgs-finance-dashboard-app-deploy-abc123.streamlit.app/
```

## ⚙️ 部署后配置

### 自定义域名（可选）
1. 在 Streamlit Cloud  dashboard
2. 点击 "Settings"
3. 找到 "Custom domain"
4. 按照指引配置 DNS

### 开启 Always On（可选，付费）
- 防止应用休眠
- $5/月

### 设置 secrets（如果需要 API key）
1. 在 Streamlit Cloud dashboard
2. 点击 "Secrets"
3. 添加密钥（如 Yahoo Finance API key）

## 🐛 故障排查

### 部署失败

**错误：ModuleNotFoundError**
```
解决方案：检查 requirements.txt 格式
- 每行一个包
- 不要有中文注释
- 使用 >= 而不是 ==
```

**错误：File not found**
```
解决方案：
- 确认主文件名正确（app_deploy.py）
- 确认文件已上传到 GitHub
- 检查大小写（Linux 区分大小写）
```

**错误：Timeout**
```
解决方案：
- 第一次部署慢是正常的
- 等待 5-10 分钟
- 如果超过 10 分钟，查看部署日志
```

### 运行正常但数据加载失败

**可能原因**：
1. Yahoo Finance API 限制
2. 网络问题
3. 股票代码错误

**解决方案**：
- 稍后重试
- 更换股票代码
- 检查时间范围（不要选"全部"）

## 📊 性能优化

### 减少加载时间
1. 使用 `@st.cache_data` 装饰器
2. 限制默认数据量（建议默认 1 年）
3. 避免在页面加载时运行耗时操作

### 减少内存使用
1. 及时删除不用的 DataFrame
2. 使用 `del` 释放内存
3. 避免加载过多历史数据
